<objective>
Create the Phase 1 database foundation before any dependent behavior is implemented.

Purpose: remove ordering risk by adding schema, migrations, and migration checks first.
Output: one explicit SQL migration, matching Drizzle schema objects, and focused migration assertions.
</objective>

<threat_model>
High severity threats block completion:
- T-01 schema drift: SQL migrations and Drizzle schema disagree, causing runtime failures or partial deploys.
- T-02 unsafe session persistence: persisted SQLite/S3 snapshots continue to rely on raw reusable bearer tokens.
- T-03 unbounded auth data: login and email-change challenge tables lack indexes or cleanup paths.
- T-04 duplicate active emails: application checks cannot be implemented consistently because schema lacks required fields.
- T-05 missing rate-limit persistence: abuse controls disappear after restart and cannot be inspected by admins.
</threat_model>
