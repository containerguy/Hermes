<objective>
Wire Phase 1 auth, profile, and invite backend capabilities into the existing React UI.

Purpose: make the hardened backend usable for LAN operators and users without starting the Phase 5 frontend extraction.
Output: CSRF-aware client requests, German error mapping, profile/device/email UI, and invite lifecycle UI. (Admin rate-limit operations UI is split into 01-08.)
</objective>

<threat_model>
High severity threats block completion:
- T-01 frontend bypass: authenticated mutations fail in production because the client does not send CSRF.
- T-02 poor lockout recovery: admins cannot clear accidental LAN rate-limit blocks through the UI.
- T-03 secret handling regression: invite UI hides codes from admins or audit/error copy encourages unsafe disclosure.
- T-04 account confusion: users cannot distinguish username login identity from mutable display name and email-change state.
- T-05 scope creep: Phase 1 UI work accidentally becomes broad component extraction.
</threat_model>
