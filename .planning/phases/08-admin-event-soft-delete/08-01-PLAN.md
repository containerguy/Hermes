# Phase 08 Plan 08-01: Admin Event Soft Delete

## Objective

Implement an **admin-only** soft-delete for events that are **already** `archived` or `cancelled`, ensure deleted events are not listed on the event board for any role, and write an audit log entry with safe metadata.

## Tasks

### Task 1 — DB support (soft-delete columns + migration)

- Add `deleted_at` and `deleted_by_user_id` to `game_events` via a new SQL migration.
- Update Drizzle schema (`src/server/db/schema.ts`) accordingly.

### Task 2 — Backend behavior (admin endpoint + listing filter)

- Add an admin endpoint under `/api/admin` to soft-delete an event.
- Enforce:
  - authenticated admin only
  - CSRF required
  - event must exist and not already be deleted
  - event status must be `archived` or `cancelled`
- Record `audit_logs` entry (`action: event.soft_delete`) with safe metadata.
- Exclude soft-deleted events from `GET /api/events` (and treat them as not-found for per-event mutations).

### Task 3 — Client UX (admin-only delete action)

- Add an admin-only UI action to soft-delete events when the event is `archived` or `cancelled`.
- After deletion, refresh the event list; deleted event must disappear.

### Task 4 — Tests + verification

- Add/extend tests to cover:
  - authorization (non-admin forbidden)
  - guardrails (status must be archived/cancelled)
  - visibility (deleted events not listed)
  - audit log entry written with expected action + safe metadata
- Run:
  - `npm test`
  - `npm run build`

## Done Criteria

- Admin can soft-delete cancelled/archived events; other statuses are rejected.
- Soft-deleted events do not appear on the event board for any role.
- Audit log contains an `event.soft_delete` entry with safe metadata.
- Test suite and build pass.
